# Engineering Process Report (EPR) — BookShelf-EPR

## Overview
This project is a student practice for understanding software development, versioning, and library responsibilities.

## Development Steps
1. Setup backend and frontend folders
2. Create basic API using FastAPI
3. Create simple frontend page
4. Add test cases
5. Track bugs and version releases

## Bugs & Fixes
- v0.1 → basic API works, frontend minimal
- v0.2 → minor bug fixes, added basic tests
- v1.0 → more stable, folder structure finalized

## Release Notes
- Each version includes small improvements to demonstrate versioning.
